# 设计方案：煎饼和墨汁儿轮询 task-publish-repo

## 1. 工作流程

```
煎饼/墨汁儿 (Heartbeat 1分钟)
  ↓
读取 task-publish-repo 的 task-config.json
  ↓
获取 repo + branch
  ↓
在 /workspace 中 clone/pull 开发仓库
  ↓
读取 milestone.md
  ↓
根据各自状态工作
```

### 关键说明

- **轮询时机**：只在 agent 空闲时触发（OpenClaw Heartbeat 机制限制）
- **正在开发时**：不轮询，开发完成后自然触发
- **仓库管理**：如果 repo/branch 变化则重新 clone，否则只 pull

## 3. 煎饼状态机

| 状态 | 说明 |
|-----|------|
| 待开发 | 等待任务，监控中 |
| 开发中 | 正在开发 |
| 开发完成 | 待一轮测试 |
| 修复中 | 第N轮修复中（N=1,2,3...） |
| 待测试 | 等待墨汁儿测试 |
| 可归档 | 测试通过，等待归档 |

### 状态流转

```
待开发 → 开发中 → 开发完成 → 待测试 → 修复中 → 待测试 → ... → 可归档
```

---

## 4. 墨汁儿状态机

| 状态 | 说明 |
|-----|------|
| 待开发 | 等待任务，监控中 |
| 用例开发中 | 编写测试用例中 |
| 用例完成 | 等待开发完成 |
| 测试中 | 第N轮测试中（N=1,2,3...） |
| 测试完成 | 第N轮测试完成，等待修复 |
| 测试通过 | 全部测试通过 |

### 状态流转

```
待开发 → 用例开发中 → 用例完成 → 测试中 → 测试完成 → 测试中 → ... → 测试通过
```

---

## 5. 轮询逻辑

### 煎饼 Heartbeat 逻辑

```
1. fetch + pull task-publish-repo (master)
2. 读取 task-config.json 获取 repo + branch
3. 如果 repo/branch 变化:
   - 删除旧仓库目录
   - clone 新仓库
   否则:
   - pull 最新代码
4. 读取 milestone.md
5. 根据开发状态执行:
   - 待开发: 开始开发 → 状态=开发中
   - 开发中: 继续开发
   - 待测试: 继续等待（不操作）
   - 修复中: 继续修复 → 状态=待测试
   - 可归档: 执行归档操作
```

### 墨汁儿 Heartbeat 逻辑

```
1. fetch + pull task-publish-repo (master)
2. 读取 task-config.json 获取 repo + branch
3. 如果 repo/branch 变化:
   - 删除旧仓库目录
   - clone 新仓库
   否则:
   - pull 最新代码
4. 读取 milestone.md
5. 根据测试状态执行:
   - 待开发: 开始编写用例 → 状态=用例开发中
   - 用例开发中: 继续编写用例 → 状态=用例完成
   - 用例完成: 等待开发完成（不操作）
   - 测试中: 继续测试 → 状态=测试完成
   - 测试完成: 等待修复（不操作）
   - 测试通过: 不操作
```

## 6. 触发条件细化

### 煎饼触发条件

| 当前状态 | 触发动作 | 下一状态 |
|---------|---------|---------|
| 待开发 | 检测到新任务 | 开发中 |
| 开发中 | 完成开发 | 开发完成 |
| 修复中 | 完成修复 | 待测试 |
| 待测试 | 煎饼不操作（等待墨汁儿） | - |
| 可归档 | 执行归档 | - |

### 墨汁儿触发条件

| 当前状态 | 触发动作 | 下一状态 |
|---------|---------|---------|
| 待开发 | 开始编写测试用例 | 用例开发中 |
| 用例开发中 | 用件编写完成 | 用例完成 |
| 用例完成 | 检测到煎饼"开发完成" | 测试中 |
| 测试中 | 完成测试 | 测试完成 |
| 测试完成 | 检测到煎饼"修复中" | 测试中 |
| 测试完成 | 煎饼不再修复 | 测试通过 |

## 8. milestone.md 模板

### 完整模板

```markdown
# Milestone: {version}

## 状态: {总状态}

## 任务列表 (刚子负责)
- [ ] 任务1: 描述xxx
- [ ] 任务2: 描述yyy

---

## 开发状态 (煎饼负责)
- **状态**: 待开发
- **当前分支**: {branch}
- **开发记录**:
  - [2026-03-15 10:00] 收到任务

---

## 测试状态 (墨汁儿负责)
- **状态**: 待开发
- **测试记录**:
  - [2026-03-15 10:00] 收到任务
```

### 煎饼状态值

- `待开发` - 等待任务，监控中
- `开发中` - 正在开发
- `开发完成` - 待一轮测试
- `修复中` - 第N轮修复中
- `待测试` - 等待墨汁儿测试
- `可归档` - 测试通过，等待归档

### 墨汁儿状态值

- `待开发` - 等待任务，监控中
- `用例开发中` - 编写测试用例中
- `用例完成` - 等待开发完成
- `测试中` - 第N轮测试中
- `测试完成` - 第N轮测试完成，等待修复
- `测试通过` - 全部测试通过

### 更新记录格式

**开发记录**:
```markdown
- [2026-03-15 10:00] 收到任务，开始开发
- [2026-03-15 10:30] M1: 创建用户模型
- [2026-03-15 11:00] M2: 实现注册接口
- [2026-03-15 11:30] 开发完成，等待测试
- [2026-03-15 14:00] 收到测试反馈，开始修复
- [2026-03-15 14:30] 修复完成，等待二轮测试
```

**测试记录**:
```markdown
- [2026-03-15 10:00] 收到任务
- [2026-03-15 10:30] 开始编写测试用例
- [2026-03-15 11:00] 测试用例完成，等待开发完成
- [2026-03-15 12:00] 煎饼开发完成，开始测试
- [2026-03-15 12:30] 一轮测试完成，发现问题：
  - 问题1: 密码未加密
  - 问题2: 缺少输入验证
- [2026-03-15 15:00] 煎饼已修复，开始二轮测试
- [2026-03-15 15:30] 二轮测试通过 ✅
```
