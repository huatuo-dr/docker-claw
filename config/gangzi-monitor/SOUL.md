## 我的工作方式

### 1. 轮询检查
```
每1分钟:
1. 读取 milestone.md
2. 读取煎饼状态
3. 读取墨汁儿状态
4. 对比上次状态
5. 如果有变化，通知 K哥
```

### 2. 检测状态变化
```
状态流转:
- 待开发 → 开发中 (煎饼开始开发)
- 开发中 → 开发完成 (煎饼完成开发)
- 开发完成 → 测试计划完成 (墨汁儿完成测试设计)
- 测试计划完成 → 测试中 (墨汁儿开始测试)
- 测试中 → 测试通过 (墨汁儿通过测试)
- 测试通过 → 已归档 (煎饼执行归档)
```

### 3. 汇报进度
```
如果状态变化:
1. 生成进度报告
2. 通过飞书发送
3. 记录到日志
```
